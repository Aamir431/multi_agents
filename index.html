üß† Multi-Agent Workflow

The system's power lies in its orchestrated pipeline, where agents execute tasks sequentially, building upon the work of their predecessors. This structured collaboration ensures a comprehensive and reliable analysis.

1. Data Loading Agent

Initiates the process by ingesting the raw CSV dataset, ensuring data is correctly loaded into the system.

2. Preprocessing Agent

Performs the critical tasks of data cleaning, handling missing values, encoding categorical features, and scaling numerical data.

3. EDA Agent

Generates a deep Exploratory Data Analysis (EDA) report with statistical summaries and rich visualizations to uncover underlying patterns.

4. Modeling Agent

Applies machine learning algorithms to the prepared data, training a model and evaluating its performance with key metrics.

üõ†Ô∏è How It Works

The system is launched via the `main.py` script, which initializes and runs the `Orchestrator`.

  1. The `Orchestrator` defines the sequence of agents to run.
  2. It creates a shared `PipelineState` object to pass data between agents.
  3. Using `asyncio`, it executes each agent in order, waiting for one to complete before starting the next.
  4. Each agent performs its task‚Äîloading, cleaning, analyzing, or modeling‚Äîand updates the shared state.
  5. The process culminates in the generation of detailed reports, providing a full summary of the data's story.
